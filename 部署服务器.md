驴行为检测项目服务器部署指南（Ubuntu 22.04）

适用环境：

系统：Ubuntu 22.04
项目目录：/var/www/realtime-detector
后端：FastAPI + Uvicorn
前端：Vue + Vite
进程守护：systemd
反向代理：Nginx
服务器公网访问：http://你的服务器IP
一、部署目标

最终效果：

后端服务运行在 127.0.0.1:8080
前端静态资源由 Nginx 提供
/api、/health、/ws 由 Nginx 转发给后端
摄像头启用后，服务器能正常拉取萤石 https://...flv 流并进行检测
浏览器直接访问服务器 IP 即可使用

七、修改后端 Linux 配置

编辑：

vim /var/www/realtime-detector/modules/config.py

建议改成下面这样：

from pathlib import Path

BASE_DIR = Path(__file__).resolve().parents[1]

# ------- 模型/输入 -------
IMG_SIZE = (640, 640)
CLASSES = ("standing", "mating", "lying")
NUM_CLASSES = len(CLASSES)
CONF_THRES = 0.25
IOU_THRES = 0.45

# ------- 事件检测参数 -------
MATING_EVENT_MIN_DURATION = 6
MATING_CONF_THRES = 0.4

# Linux 服务器使用 .pt 模型
PT_MODEL_PATH = str(BASE_DIR / "models" / "0710-best-YOLO.pt")

# ------- 本地流水线 / WS -------
MAX_FPS = 10
FRAME_INTERVAL = 2
JPEG_QUALITY = 60
QUEUE_MAX = 2

# ------- 输入源默认 -------
RTSP_URL = "rtsp://user:pass@ip:port/xxx"
VIDEO_PATH = str(BASE_DIR / "data" / "demo.mp4")
IMAGE_PATH = str(BASE_DIR / "data" / "111.png")

# ------- MPV Pipe -------
MPV_DIR = "/usr/bin"
MPV_EXE = "/usr/bin/mpv"

MPV_PIPE_W = 800
MPV_PIPE_H = 600
MPV_PIPE_FPS = 8

# ------- 颜色（BGR） -------
COLOR_STANDING = (0, 255, 0)
COLOR_MATING = (0, 0, 255)
COLOR_LYING = (255, 0, 0)
说明

这一步必须改，因为原始配置里通常会混有 Windows 路径，例如：

E:\...0710-best-YOLO.pt
D:\...\mpv.exe

Linux 下会直接失败。

九、前端改为“同域访问”

这是上线时非常容易漏掉的坑。
如果前端仍然写死 localhost:8080，浏览器访问服务器 IP 时，会请求访问者自己电脑的 localhost，接口当然会错。

进入前端目录：

cd /var/www/realtime-detector/vue-frontend
1）修改 src/composables/useApi.ts
const API_BASE_URL = '';
2）修改 src/composables/useWebSocket.ts
const WS_BASE_URL = `${window.location.protocol === 'https:' ? 'wss' : 'ws'}://${window.location.host}`;
3）修改 store 中 axios baseURL

以下文件统一改成：

axios.defaults.baseURL = '';

涉及：

src/stores/event.ts
src/stores/camera.ts
src/stores/barn.ts
src/stores/pen.ts
4）修改截图地址

src/components/EventRecord.vue 中如果有：

return `http://localhost:8080${screenshotPath}`;
return `http://localhost:8080/${screenshotPath}`;

要改成：

return `${window.location.origin}${screenshotPath}`;
return `${window.location.origin}/${screenshotPath}`;
5）检查是否还有硬编码
grep -R "localhost:8080" -n src

理想情况是没有任何输出。

十一、配置 systemd 让后端常驻

先停止手工运行的 uvicorn（在前台窗口按 Ctrl+C），然后创建 service 文件：

cat >/etc/systemd/system/realtime-detector.service <<'EOF'
[Unit]
Description=Realtime Detector Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/var/www/realtime-detector
Environment="PYTHONUNBUFFERED=1"
Environment="PATH=/var/www/realtime-detector/donkey_env/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
ExecStart=/var/www/realtime-detector/donkey_env/bin/python -m uvicorn modules.main:app --host 0.0.0.0 --port 8080
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

然后：

systemctl daemon-reload
systemctl enable --now realtime-detector
systemctl status realtime-detector --no-pager -l
说明

一定要加：

Environment="PYTHONUNBUFFERED=1"

否则检测线程里的 print 日志可能缓冲，看不到实时排障信息。

十二、配置 Nginx

创建配置：

cat >/etc/nginx/sites-available/realtime-detector <<'EOF'
server {
    listen 80;
    server_name _;

    root /var/www/realtime-detector/vue-frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /upload/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }

    location /health {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
EOF

启用配置：

ln -sf /etc/nginx/sites-available/realtime-detector /etc/nginx/sites-enabled/realtime-detector
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
systemctl enable nginx

十五、摄像头检测不工作时的排查思路

你这次最重要的排障经验就是这一部分。

第一步：看请求有没有真正触发
journalctl -u realtime-detector --since "20 min ago" --no-pager | egrep "POST /api/camera-configs|PATCH /api/camera-configs/.*/toggle|GET /api/camera-configs"

如果只有 GET /api/...，没有 POST/PATCH，说明前端没真正触发启用检测。

第二步：看数据库配置是否启用
sqlite3 data/farm.db "select id,camera_id,enable,barn_id,pen_id,start_time,end_time from camera_configs order by id;"

你这次实际就是：

1|5-2|1|1|1|09:00|19:00
2|5-3|1|1|1|09:00|19:00

说明配置已启用。

第三步：检查摄像头表结构与字段

你这次的实际表结构是：

cameras
id INTEGER PRIMARY KEY AUTOINCREMENT,
camera_id TEXT NOT NULL UNIQUE,
pen_id INTEGER NOT NULL,
barn_id INTEGER NOT NULL,
flv_url TEXT NOT NULL
camera_configs
id INTEGER PRIMARY KEY AUTOINCREMENT,
camera_id TEXT NOT NULL,
flv_url TEXT NOT NULL,
barn_id INTEGER NOT NULL,
pen_id INTEGER NOT NULL,
enable INTEGER DEFAULT 1

这意味着：

业务摄像头标识是 camera_id，值类似 5-2
流地址字段是 flv_url
启动检测最稳的方式是直接从数据库里用 flv_url，而不是依赖 config.CAMERA_FLV_URLS。
第四步：修掉旧版代码依赖 CAMERA_FLV_URLS

你这次已经修过：

streams.py 中数据库优先反查 flv_url/camera_id/pen_id/barn_id
CAMERA_FLV_URLS 只作为可选兜底
main.py 修复快速 toggle 竞态
启动前 flv_url.strip()
关键日志 flush=True

这一步很重要，否则检测线程会在启动时直接崩。

十六、如果检测线程启动后 mpv 很快退出

你这次最终定位到的核心问题是：

不是 Python 逻辑，而是服务器旧版 mpv/ffmpeg 无法解码萤石的 H.265 FLV 流。

旧服务日志中出现过：

Video codec (c) is not implemented
vo/lavc error encoding at EOF
mpv exited rc=2
READER fps≈0, ok=False, frame=None

而你本地能跑，是因为本地版本明显更新：

Windows 本地：mpv 0.41.0，FFmpeg 7.x
服务器原始环境：mpv 0.34.1，FFmpeg 4.4.2

3）验证新版 ffmpeg 能读流

先从数据库取流地址：

URL=$(sqlite3 /var/www/realtime-detector/data/farm.db "select flv_url from cameras where camera_id='5-2';" | xargs)
echo "[$URL]"

再测试：

/opt/ffmpeg/bin/ffmpeg -i "$URL" -t 5 -f null -

你这次实际已经验证成功，新版 FFmpeg 能正常识别：

输入 flv
视频 hevc (Main)
输出成功处理 125 帧。

这一步非常关键，因为它直接证明：

问题根因就是服务器旧版 ffmpeg/mpv 太旧，不支持这个流。

十八、给服务器升级新版 mpv（关键）
1）先升级 Meson

Ubuntu 22.04 自带 meson 0.61.2 太老，mpv-build 需要 >=1.3.0。

python3 -m pip install --upgrade meson ninja
export PATH=/root/.local/bin:$PATH
meson --version

如果要长期生效：

echo 'export PATH=/root/.local/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
2）编译安装 mpv
cd /usr/local/src
git clone https://github.com/mpv-player/mpv-build.git
cd mpv-build

写 mpv_options：

echo "--prefix=/opt/mpv" > mpv_options
echo "-Dlibmpv=true" >> mpv_options

注意：不要写旧的 --enable-libmpv-shared，meson 不认。

写 ffmpeg_options：

cat > ffmpeg_options <<'EOF'
--prefix=/opt/mpv
--enable-gpl
--enable-version3
--enable-shared
--disable-debug
--enable-gnutls
--enable-libx264
--enable-libx265
--enable-libvpx
--enable-libopus
--enable-libmp3lame
--enable-libvorbis
--enable-libtheora
--enable-libass
--enable-libfreetype
--enable-libfribidi
--enable-libharfbuzz
--enable-libsrt
--enable-libssh
--enable-libsoxr
EOF

清理并构建：

rm -rf mpv/build
./build
./install
echo '/opt/mpv/lib' >/etc/ld.so.conf.d/opt-mpv.conf
ldconfig
/opt/mpv/bin/mpv --version

你这次实际安装成功，版本已到：

mpv v0.41.0-...

并绑定的是新版 FFmpeg。

十九、把项目切到新版 mpv

修改配置：

sed -i 's|^MPV_EXE *=.*|MPV_EXE = "/opt/mpv/bin/mpv"|' /var/www/realtime-detector/modules/config.py
grep -n "MPV_EXE" /var/www/realtime-detector/modules/config.py

确认已经变成：

MPV_EXE = "/opt/mpv/bin/mpv"

二十二、最终结论

这套项目在 Ubuntu 22.04 上的部署核心有两层：

基础部署层
Python 虚拟环境
Linux 路径修正
前端同域访问修复
npm run build
systemd
Nginx
摄像头检测层
不要再依赖 config.CAMERA_FLV_URLS
优先从数据库读取 camera_configs / cameras 里的 flv_url
服务器旧版 mpv 0.34.1 + ffmpeg 4.4.2 不足以处理你的萤石 H.265 FLV 流
必须升级到新版 FFmpeg 8.1 + mpv 0.41 才能与本地环境能力接近，最终解决检测线程启动后 mpv 立即退出的问题。
