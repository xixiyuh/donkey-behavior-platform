## 核心优化点

### 1. 分离检测和展示（最关键）
```
后台模式（无WebSocket）：
  reader → infer → event_process/screenshot/save
  ✅ 不画框、不编码、不广播

展示模式（有WebSocket）：
  reader → infer → event_process/screenshot/save
                → draw_box → encode → broadcast
```

### 2. 版本机制（避免重复）
```
frame_version: 读帧时++
  ↓
infer仅在变化时处理
  ↓
result_version: 推理时++
  ↓
WebSocket仅在display_version变化时发送
  ↓
统一编码：一次编码，多客户端共享
```

### 3. 事件驱动（避免轮询）
```
new_frame_event: reader读到帧时set()
                 infer.wait()时立即唤醒
                 无帧时完全阻塞

subscribers_changed_event: 订阅者变化时set()
                           可用于模式切换
```

## 代码示例

### 示例1：后台检测模式下的infer_loop

```python
# 仅当frame_version变化时才推理（避免重复）
if current_frame_version == last_frame_version:
    new_frame_event.wait()  # 完全阻塞，无轮询
    continue

# 推理
detection_result = detector.infer_once(frame)
result_version += 1

# 后台业务（必须保留）
process_detection(detection_result)

# 展示（仅当有订阅者）
if ws_subscriber_count > 0:  # ← 关键：条件检查
    display_frame = draw_boxes(frame, detection_result)
    display_version += 1
```

### 示例2：WebSocket版本驱动发送

```python
# 版本未变，不发送
if current_display_version == last_sent_version:
    await asyncio.sleep(0.01)
    continue

# 版本变化，统一编码并发送
jpeg_bytes, version = session.encode_and_cache_jpeg()  # ← 统一编码
await ws.send_text(base64.b64encode(jpeg_bytes))
last_sent_version = version
```

### 示例3：多客户端共享编码

```python
# 旧方式：N客户端，N次编码 ❌
for client in ws_clients:
    jpeg = cv2.imencode(frame)  # 每个客户端都编码
    send(client, jpeg)

# 新方式：1次编码，N客户端共享 ✅
jpeg = session.encode_and_cache_jpeg()  # 仅编码1次
for client in ws_clients:
    send(client, jpeg)  # 共享同一份JPEG
```
