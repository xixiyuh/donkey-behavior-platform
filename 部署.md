# 驴行为检测平台部署说明

本文档说明如何配置并部署本项目的两套环境：

- 本机开发环境：`local`
- 服务器生产环境：`production`

项目结构：

- 后端：FastAPI，入口为 `run.py`
- 前端：Vue3 + Vite，目录为 `vue-frontend`
- 数据库：MySQL
- 生产部署：Nginx 反向代理前端静态文件、HTTP API、WebSocket

## 1. 后端环境变量

后端配置从项目根目录 `.env` 或系统环境变量读取。先复制示例文件：

```powershell
copy .env.example .env
```

Linux 服务器：

```bash
cp .env.example .env
```

### 本机开发 `.env`

```env
APP_ENV=local
APP_HOST=127.0.0.1
APP_PORT=8080
DEBUG=true
LOG_LEVEL=INFO

DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=farm_user
DB_PASSWORD=你的本机数据库密码
DB_NAME=farm_db
DB_CHARSET=utf8mb4

ALLOWED_ORIGINS=http://localhost:5173,http://127.0.0.1:5173

UPLOAD_DIR=uploads
IMAGE_MAX_UPLOAD_SIZE=10485760
VIDEO_MAX_UPLOAD_SIZE=524288000
```

### 服务器生产 `.env`

```env
APP_ENV=production
APP_HOST=127.0.0.1
APP_PORT=8080
DEBUG=false
LOG_LEVEL=INFO

DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=farm_user
DB_PASSWORD=你的生产数据库密码
DB_NAME=farm_db
DB_CHARSET=utf8mb4

ALLOWED_ORIGINS=https://你的域名

UPLOAD_DIR=/var/www/realtime-detector/uploads
IMAGE_MAX_UPLOAD_SIZE=10485760
VIDEO_MAX_UPLOAD_SIZE=524288000
```

注意：

- 不要把真实 `.env` 提交到 Git。
- 生产环境建议 `APP_HOST=127.0.0.1`，只让 Nginx 访问后端。
- `DB_PASSWORD` 不再写在代码里，必须在 `.env` 或 systemd 环境文件中配置。

## 2. 数据库准备

在 MySQL 中创建数据库和专用用户。示例：

```sql
CREATE DATABASE farm_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'farm_user'@'127.0.0.1' IDENTIFIED BY '请换成强密码';
GRANT ALL PRIVILEGES ON farm_db.* TO 'farm_user'@'127.0.0.1';
FLUSH PRIVILEGES;
```

如果你的 MySQL 使用 `localhost` socket 连接，也可以创建：

```sql
CREATE USER 'farm_user'@'localhost' IDENTIFIED BY '请换成强密码';
GRANT ALL PRIVILEGES ON farm_db.* TO 'farm_user'@'localhost';
FLUSH PRIVILEGES;
```

后端启动时会执行建表逻辑，不需要手动建业务表。

## 4. 本机开发启动

### 后端

确认根目录 `.env` 已配置好，然后运行：

```powershell
python run.py
```

后端默认地址：

```text
http://127.0.0.1:8080
```

### 前端

进入前端目录，复制本机开发环境变量示例：

```powershell
cd vue-frontend
copy .env.local.example .env.local
npm install
npm.cmd run dev
```

`vue-frontend/.env.local` 内容：

```env
VITE_API_BASE_URL=http://127.0.0.1:8080
VITE_WS_BASE_URL=ws://127.0.0.1:8080
```

访问：

```text
http://localhost:5173
```

## 5. 生产前端构建

进入前端目录：

```bash
cd vue-frontend
cp .env.production.example .env.production
npm install
npm run build -- --mode production
```

Windows PowerShell：

```powershell
cd vue-frontend
copy .env.production.example .env.production
npm install
npm.cmd run build -- --mode production
```

生产环境建议保持：

```env
VITE_API_BASE_URL=
VITE_WS_BASE_URL=
```

这样前端会使用当前域名的相对路径访问：

- API：`/api`
- 上传：`/upload`
- WebSocket：`/ws`

由 Nginx 统一反向代理到后端。

## 6. 生产目录建议

示例部署目录：

```text
/var/www/realtime-detector
├── backend
├── modules
├── vue-frontend
│   └── dist
├── uploads
├── static
├── logs
├── run.py
├── requirements.txt
└── .env
```

确保运行用户对这些目录有权限：

```bash
mkdir -p /var/www/realtime-detector/uploads
mkdir -p /var/www/realtime-detector/static
mkdir -p /var/www/realtime-detector/logs
chown -R www-data:www-data /var/www/realtime-detector/uploads /var/www/realtime-detector/static /var/www/realtime-detector/logs
```

如果 systemd 使用其他用户，请把 `www-data` 换成实际用户。

## 7. systemd 服务示例

创建文件：

```bash
sudo nano /etc/systemd/system/realtime-detector.service
```

示例内容：

```ini
[Unit]
Description=Realtime Donkey Behavior Detector
After=network.target mysql.service

[Service]
Type=simple
WorkingDirectory=/var/www/realtime-detector
EnvironmentFile=/var/www/realtime-detector/.env
ExecStart=/var/www/realtime-detector/venv/bin/python run.py
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable realtime-detector
sudo systemctl start realtime-detector
sudo systemctl status realtime-detector
```

查看日志：

```bash
journalctl -u realtime-detector -f
```

## 8. Nginx 配置示例

示例：

```nginx
server {
    listen 80;
    server_name 你的域名;

    root /var/www/realtime-detector/vue-frontend/dist;
    index index.html;

    client_max_body_size 500M;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /upload {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /static {
        alias /var/www/realtime-detector/static;
        expires 30d;
    }

    location /health {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600s;
    }
}
```

启用配置后：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 9. 端口暴露建议

生产环境：

- 对外开放：`80`、`443`
- 不对外开放：`8080`
- 后端 `APP_HOST=127.0.0.1`

如果使用防火墙：

```bash
sudo ufw allow 80
sudo ufw allow 443
sudo ufw deny 8080
```

## 11. 常见问题

### 1. 数据库报 Access denied

检查：

- `.env` 中 `DB_USER`
- `.env` 中 `DB_PASSWORD`
- MySQL 是否给该用户授权
- `DB_HOST` 是 `127.0.0.1` 还是 `localhost`

### 2. 前端请求 API 404

生产环境检查 Nginx：

- `/api` 是否 proxy 到 `http://127.0.0.1:8080`
- 后端服务是否正在运行
- 前端 `.env.production` 是否保持相对路径配置

### 3. WebSocket 连接失败

检查 Nginx `/ws`：

- `proxy_set_header Upgrade $http_upgrade;`
- `proxy_set_header Connection "upgrade";`
- `proxy_read_timeout` 是否足够长

### 4. 上传大文件失败

同时检查：

- 后端 `.env` 中 `VIDEO_MAX_UPLOAD_SIZE`
- Nginx `client_max_body_size`
- `UPLOAD_DIR` 是否有写入权限

## 12. 部署顺序总结

1. 准备 MySQL 数据库和用户。
2. 复制并填写根目录 `.env`。
3. 安装后端依赖。
4. 复制并填写前端 `.env.production`。
5. 执行前端生产构建。
6. 配置 systemd 启动后端。
7. 配置 Nginx 指向 `vue-frontend/dist` 并反代 `/api`、`/upload`、`/ws`。
8. 检查 `systemctl status`、`nginx -t` 和页面访问。
