# 驴行为检测平台部署说明

本文档说明如何配置并部署本项目的两套环境：

- 本机开发环境：`local`
- 服务器生产环境：`production`

项目结构：

- 后端：FastAPI，入口为 `run.py`
- 前端：Vue3 + Vite，目录为 `vue-frontend`
- 数据库：MySQL
- 生产部署：Nginx 反向代理前端静态文件、HTTP API、WebSocket

## 1. 后端环境变量

后端配置从项目根目录 `.env` 或系统环境变量读取。先复制示例文件：

```powershell
copy .env.example .env
```

Linux 服务器：

```bash
cp .env.example .env
```

### 本机开发 `.env`

```env
APP_ENV=local
APP_HOST=127.0.0.1
APP_PORT=8080
DEBUG=true
LOG_LEVEL=INFO

DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=farm_user
DB_PASSWORD=你的本机数据库密码
DB_NAME=farm_db
DB_CHARSET=utf8mb4

ALLOWED_ORIGINS=http://localhost:5173,http://127.0.0.1:5173

UPLOAD_DIR=uploads
IMAGE_MAX_UPLOAD_SIZE=10485760
VIDEO_MAX_UPLOAD_SIZE=524288000
```

### 服务器生产 `.env`

```env
APP_ENV=production
APP_HOST=127.0.0.1
APP_PORT=8080
DEBUG=false
LOG_LEVEL=INFO

DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=farm_user
DB_PASSWORD=你的生产数据库密码
DB_NAME=farm_db
DB_CHARSET=utf8mb4

ALLOWED_ORIGINS=https://你的域名

UPLOAD_DIR=/var/www/realtime-detector/uploads
IMAGE_MAX_UPLOAD_SIZE=10485760
VIDEO_MAX_UPLOAD_SIZE=524288000
```

注意：

- 不要把真实 `.env` 提交到 Git。
- 生产环境建议 `APP_HOST=127.0.0.1`，只让 Nginx 访问后端。
- `DB_PASSWORD` 不再写在代码里，必须在 `.env` 或 systemd 环境文件中配置。

## 2. 数据库准备

在 MySQL 中创建数据库和专用用户。示例：

```sql
CREATE DATABASE farm_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'farm_user'@'127.0.0.1' IDENTIFIED BY '请换成强密码';
GRANT ALL PRIVILEGES ON farm_db.* TO 'farm_user'@'127.0.0.1';
FLUSH PRIVILEGES;
```

如果你的 MySQL 使用 `localhost` socket 连接，也可以创建：

```sql
CREATE USER 'farm_user'@'localhost' IDENTIFIED BY '请换成强密码';
GRANT ALL PRIVILEGES ON farm_db.* TO 'farm_user'@'localhost';
FLUSH PRIVILEGES;
```

后端启动时会执行建表逻辑，不需要手动建业务表。

## 7. systemd 服务示例

创建文件：

```bash
sudo nano /etc/systemd/system/realtime-detector.service
```

示例内容：

```ini
[Unit]
Description=Realtime Donkey Behavior Detector
After=network.target mysql.service

[Service]
Type=simple
WorkingDirectory=/var/www/realtime-detector
EnvironmentFile=/var/www/realtime-detector/.env
ExecStart=/var/www/realtime-detector/venv/bin/python run.py
Restart=always
RestartSec=5
User=www-data
Group=www-data

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable realtime-detector
sudo systemctl start realtime-detector
sudo systemctl status realtime-detector
```

查看日志：

```bash
journalctl -u realtime-detector -f
```

## 8. Nginx 配置示例

示例：

```nginx
server {
    listen 80;
    server_name 你的域名;

    root /var/www/realtime-detector/vue-frontend/dist;
    index index.html;

    client_max_body_size 500M;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /upload {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /static {
        alias /var/www/realtime-detector/static;
        expires 30d;
    }

    location /health {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600s;
    }
}
```

启用配置后：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 11. 常见问题

### 1. 数据库报 Access denied

检查：

- `.env` 中 `DB_USER`
- `.env` 中 `DB_PASSWORD`
- MySQL 是否给该用户授权
- `DB_HOST` 是 `127.0.0.1` 还是 `localhost`

### 2. 前端请求 API 404

生产环境检查 Nginx：

- `/api` 是否 proxy 到 `http://127.0.0.1:8080`
- 后端服务是否正在运行
- 前端 `.env.production` 是否保持相对路径配置

### 3. WebSocket 连接失败

检查 Nginx `/ws`：

- `proxy_set_header Upgrade $http_upgrade;`
- `proxy_set_header Connection "upgrade";`
- `proxy_read_timeout` 是否足够长

### 4. 上传大文件失败

同时检查：

- 后端 `.env` 中 `VIDEO_MAX_UPLOAD_SIZE`
- Nginx `client_max_body_size`
- `UPLOAD_DIR` 是否有写入权限

## 12. 部署顺序总结

1. 准备 MySQL 数据库和用户。
2. 复制并填写根目录 `.env`。
3. 安装后端依赖。
4. 复制并填写前端 `.env.production`。
5. 执行前端生产构建。
6. 配置 systemd 启动后端。
7. 配置 Nginx 指向 `vue-frontend/dist` 并反代 `/api`、`/upload`、`/ws`。
8. 检查 `systemctl status`、`nginx -t` 和页面访问。

## 13. 服务器硬件与生产部署建议

你的服务器配置为：

- CPU：Intel Core i9-14900KF，24 核 32 线程
- GPU：NVIDIA GeForce RTX 4090

这台机器非常适合做“视频流接入 + GPU 行为检测推理 + FastAPI 接口服务 + MySQL 数据存储”。建议不要只把它当普通 Web 服务器使用，而是按推理服务器来规划。

### 13.1 推荐服务分工

- CPU：负责 FastAPI、MySQL、视频解码调度、后台任务、数据清理。
- RTX 4090：专门负责 YOLO/行为识别模型推理。
- Nginx：作为唯一公网入口，负责静态前端、HTTP API 反向代理、WebSocket 反向代理。
- FastAPI：只监听 `127.0.0.1:8080`，不要直接暴露公网。
- MySQL：只监听本机或内网，不开放公网 `3306`。

推荐结构：

```text
公网用户
   |
 Nginx :80/:443
   |
   |-- 前端静态文件 vue-frontend/dist
   |
   |-- /api    -> FastAPI 127.0.0.1:8080
   |-- /upload -> FastAPI 127.0.0.1:8080
   |-- /ws     -> FastAPI WebSocket
          |
       MySQL 127.0.0.1:3306
          |
       RTX 4090 推理
```

### 13.2 生产环境后端配置建议

生产环境 `.env` 建议：

```env
APP_ENV=production
APP_HOST=127.0.0.1
APP_PORT=8080

LOG_LEVEL=INFO

DB_HOST=127.0.0.1

UPLOAD_DIR=/data/donkey/uploads
IMAGE_MAX_UPLOAD_SIZE=10485760
```

重点：

- `APP_HOST=127.0.0.1`，后端只允许本机访问。
- 公网只开放 Nginx 的 `80/443`。
- 不要开放 `8080`。
- 不要开放 MySQL 的 `3306`。
- `DEBUG=false`，避免生产环境暴露异常细节。

### 13.3 生产环境前端配置建议

- API：`/api/...`

- 不写死服务器 IP。
- 后续更换域名不用重新改代码。
- 配合 Nginx 反向代理更稳定。

### 13.5 GPU 推理并发建议

RTX 4090 性能很强，但不建议无限制开启检测流。

建议初始策略：

- 同时检测视频流先从 `4` 路开始。
- 如果视频是 1080p，可以先测试 `4` 路，再逐步增加到 `6-8` 路。
- 如果分辨率较低、模型较小，可以再增加并发。
- 模型应尽量复用，不要每个请求重复加载模型。
- 视频流断开或检测停止后，要及时释放解码和推理资源。

监控 GPU：

```bash
nvidia-smi
```

重点观察：

- `GPU-Util`
- `Memory-Usage`
- `Temperature`
- `Power Draw`

### 13.6 CPU 和散热建议

i9-14900KF 性能很强，但长时间视频解码、调度和后台任务会带来较高温度。

建议：

- 生产环境以稳定为第一优先，不建议极限超频。
- 保证机箱风道和 CPU 散热可靠。
- 长时间运行时观察 CPU 温度和系统稳定性。
- 后端服务使用 systemd 守护，异常退出后自动重启。

### 13.7 CUDA / PyTorch 版本记录

部署成功后建议记录驱动、CUDA、PyTorch 版本，后续不要随意升级。

检查命令：

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available(), torch.version.cuda)"
```

如果输出 `torch.cuda.is_available()` 为 `True`，说明 PyTorch 可以正常使用 GPU。

### 13.9 最重要的生产原则

生产环境建议遵守这几条：

1. Nginx 是唯一公网入口。
2. FastAPI 只监听 `127.0.0.1:8080`。
3. MySQL 不开放公网。
4. 前端生产构建使用相对路径，不写死服务器 IP。
5. GPU 推理限制并发，先小规模测试再逐步增加。
6. systemd 管理后端服务，确保崩溃后自动重启。
7. `DEBUG=false`，日志使用 `INFO` 或 `WARNING`。
